Week 01 · Foundations

Setup & Exploratory Data Analysis

Get a working environment, then learn to look at data before you model it.

Overview

This course is a toolbox: every week adds a method, and the real skill is knowing which one to reach for. Before we touch a single model, we spend a week on the unglamorous-but-decisive part of machine learning — getting a reproducible Python environment running, and developing an instinct for what a dataset actually looks like. Roughly 80% of real ML work happens here, in loading, cleaning, and exploring data. We'll use the Titanic dataset as our running example: small enough to hold in your head, rich enough to show every preprocessing decision that matters.

By the end of this week you can…
  • Set up an isolated Python environment and install the course requirements.
  • Load a dataset with pandas and run a 60-second “first-look” diagnostic.
  • Diagnose and treat missing values, choosing a strategy per column.
  • Encode categoricals and scale features — and explain why the choice depends on the model.
  • Summarise and visualise distributions, correlations, and outliers — and know when summary statistics lie.

The Datasaurus Dozen — why you must plot your data

Twelve datasets with identical mean, variance, and correlation — yet one is a dinosaur, one is a star, one is a set of stripes. The single most convincing argument that summary statistics are not enough. (Same idea as Anscombe's quartet, taken to the extreme.)
