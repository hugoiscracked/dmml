The Project

A semester-long thread: pick a problem early, then grow it as the toolbox grows.

How the project works

The project is where the toolbox meets a real problem. The goal is not novelty — it's rigorous application. You'll work in groups of up to four, and you start early (Week 6) — but you don't have to know everything to begin. The trick:

🧵 Your dataset travels with you

Choose a dataset and frame a problem now, with the tools you have. Then, as each new method arrives — clustering & PCA (W7), neural networks (W8), CNNs (W9), transformers (W10) — apply it to your own data. By the final week you'll have layered the most powerful tools in the course onto a problem you've lived with for six weeks. The interesting tools are still ahead of you; the project is where you'll try them.

What a strong project looks like

  • Framed first. Inputs, target, and evaluation metric defined before modeling.
  • Baseline always. Start with a stupid baseline (majority class, predict-the-mean); report how much you beat it.
  • Broad. Compare several model families across the toolbox, including at least one advanced method (boosting and/or a neural approach). A single decision tree is not a project.
  • Leak-proof. Test set untouched until the end; CV for tuning; pipelines for preprocessing.
  • Interpreted & honest. Explain why a model works (feature importance, SHAP), do real error analysis, and write an honest limitations section.

A note on tools like LLMs: they'll happily write the code. They will not do your error analysis on your data, justify your model choices, or find where your model fails — and that's exactly what the grade rewards. Expect to explain your choices in the project labs.

Find a dataset

Avoid toy sets (MNIST, Iris) and heavily Kaggle-overfit ones. Aim for >1000 rows, a clear license, and a question that matches the data.

Timeline & deliverables

  • Week 6 — Kickoff. Form your group, choose a dataset, frame the problem. Start EDA + a baseline with the tools you have.
  • Weeks 7–10 — Grow it. Each week, fold the new method into your project as you learn it.
  • Week 9 — Proposal due (pass/fail). One page: problem, dataset, evaluation metric, baseline plan.
  • Weeks 11–12 — Project labs + clinic. Dedicated in-class time; methodology deep-dive in W11.
  • End of Week 12 — Report due. ≈8–15 pages: Problem → Data → Methods → Results → Interpretation → Limitations → Conclusion. Deliberately before the exam period.
