Final-push checklist
  • Baseline reported? Show how much each model beats a stupid baseline.
  • Breadth covered? Several model families, including at least one advanced method.
  • No leakage? Test set still untouched; preprocessing inside the CV pipeline.
  • Error analysis done? Where does the model fail, and why? (Don't just report a metric.)
  • Limitations written? An honest account of what doesn't work is a sign of rigor.
  • Reproducible? Seeds set, versions pinned, code runs — graders will run it.
